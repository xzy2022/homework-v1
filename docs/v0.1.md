# UGV Path Tracking v0.1

## 功能概述
- 基于 ROS Noetic 的 catkin 工作区 `ros_ws` 已初始化，包含 `ugv_msgs` 与 `ugv_path_tracking` 两个包。
- `ugv_msgs` 提供轨迹、车辆状态、控制指令与评估指标的自定义消息，统一分布式节点的数据格式。
- `ugv_path_tracking` 实现了
  - 全局路径节点 `global_path_node`（支持圆形、8字形、正弦、CSV），周期发布期望轨迹；
  - 纯追踪、Stanley 控制器节点，独立订阅车辆状态并输出控制命令；
  - `tracking_evaluator_node` 周期性计算 RMS 误差、速度、控制平滑性等指标，用于多算法对比。
- 控制器参数以 YAML 管理（`config/pure_pursuit.yaml`、`config/stanley.yaml`），`launch/path_tracking_framework.launch` 示例用于一次性启动所有进程。

## 使用提示
1. `cd ros_ws && catkin_make` 完成编译。
2. `source devel/setup.bash`。
3. `roslaunch ugv_path_tracking path_tracking_framework.launch path_type:=circle` 启动整体框架（根据仿真模型提供 `/vehicle/state`、执行/评估接口）。
